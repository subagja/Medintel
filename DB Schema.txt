ERD Inti (Minimal tapi production-ready)
1) users (opsional jika pakai Django auth)
* user_id (PK)
* username, email, role
* created_at, last_login

Kalau pakai Django, ini biasanya auth_user + groups/permissions.

2) sources
Menyimpan sumber OSINT (RSS/site).
* source_id (PK)
* name (Kompas, Halodoc, dll)
* base_url
* rss_url (nullable)
* country_code (ID/US/…)
* is_active (bool)
* created_at

Relasi:
sources (1) -> (N) signals

3) signals (inti data crawling)
Satu baris = satu sinyal berita.
* signal_id (PK)
* source_id (FK -> sources)
* disease_tag (text / FK kalau mau normalized)
* title
* url (original)
* final_url
* published_at (datetime)
* crawled_at (datetime)
* summary (text)
* content_text (text) (boleh disimpan ringkas / hash untuk hemat)
* threat_score (int 0–100)
* status (enum: raw, triaged, validated, rejected)
* dedup_hash (unique) (hash dari title+final_url atau simhash)
* language (id/en)
* created_at

Relasi:
* signals (1) -> (N) signal_locations
* signals (1) -> (N) signal_events (opsional)
* signals (1) -> (N) annotations

4) locations (gazetteer master)
Ini versi database dari gazetteer_id.csv.
* location_id (PK)
* name (contoh: Kota Bogor)
* level (enum: country, province, kabupaten, kota, kecamatan, desa, facility)
* parent_id (FK -> locations, nullable) (hirarki: kota parent=provinsi, dst)
* country_code (ID)
* lat, lon (decimal)
* geocode_provider (Nominatim/Manual)
* geocode_quality (enum: exact, approx, centroid)
* is_active (bool)
* created_at, updated_at

Relasi:
* locations (1) -> (N) location_aliases
* locations (1) -> (N) signal_locations

5) location_aliases (alias/singkatan)
Untuk meningkatkan match lokasi.
* alias_id (PK)
* location_id (FK -> locations)
* alias_text (mis: “Kab. Bogor”, “Pemkab Bogor”, “Bgr”)
* alias_norm (indexed) (lowercase cleaned)
* created_at

6) signal_locations (hasil ekstraksi lokasi per sinyal)
Karena 1 berita bisa menyebut banyak lokasi.
* signal_location_id (PK)
* signal_id (FK -> signals)
* location_id (FK -> locations, nullable) (kalau match gazetteer)
* raw_location_text (text) (lokasi_mentah)
* method (enum: gazetteer, regex, ner, manual)
* confidence (float 0–1)
* geocode_status (enum: ok, not_found, empty, skip_noise, skip_general, net_err)
* lat, lon (decimal, nullable) (snapshot hasil geocode untuk record)
* is_primary (bool) (lokasi kejadian utama?)
* created_at

Untuk prototype awal, biasanya 1 lokasi utama cukup. Tapi desain ini siap multi-lokasi.

#ERD untuk Early Warning System (Layer Eskalasi)#
7) alerts (trigger alert)
Satu alert = hasil deteksi pola.
* alert_id (PK)
* alert_type (enum: cluster, spike, high_score, new_location, rumor)
* severity (enum: low, medium, high, critical)
* status (enum: open, investigating, closed, false_positive)
* title
* description
* created_at, updated_at
* created_by (FK -> users, nullable untuk auto)

Relasi:
* alerts (N) <-> (N) signals via alert_signals
* alerts (N) <-> (N) locations via alert_locations

8) alert_signals (link alert ke sinyal)
* id (PK)
* alert_id (FK)
* signal_id (FK)

9) alert_locations
* id (PK)
* alert_id (FK)
* location_id (FK)

10) alert_rules (konfigurasi threshold)
Biar analis bisa ubah rules tanpa ubah kode.
* rule_id (PK)
* name
* rule_type (cluster, spike, keyword, dll)
* disease_tag (nullable)
* window_hours (int)
* threshold_count (int)
* threshold_score_avg (int, nullable)
* location_level_filter (nullable)
* is_active (bool)
* created_at

#ERD untuk Workflow Intel (Validasi & Audit)#
11) annotations (catatan analis)
* annotation_id (PK)
* signal_id (FK)
* user_id (FK)
* note (text)
* label (enum: valid, noise, needs_review, confirmed_outbreak)
* created_at

12) audit_logs (jejak perubahan)
* audit_id (PK)
* user_id (FK)
* entity_type (signals/locations/alerts/…)
* entity_id
* action (create/update/delete)
* changes_json (text/json)
* created_at

Relasi Utama (Ringkas)
* sources 1—N signals
* signals 1—N signal_locations
* locations 1—N location_aliases
* locations (parent-child) via parent_id
* alerts N—N signals
* alerts N—N locations
* signals 1—N annotations
* users 1—N annotations
* users 1—N audit_logs
